こんにちは、hinemos管理人です。ここのページを見ているということは、既にhinemosのアカウントは登録していただけていると思います。ご登録ありがとうございます！

このページは、外部サイトの3x3x3目隠し M2/OP法 甘やかし編 概論(以下、「概論」と呼びます)の続きです。概論では、3BLDの試技の流れと、ナンバリングとは何かについて説明しました。この記事では遂に、具体的な分析のやり方の説明に入ります！ そして、概論の最後で宣言した通り、説明はさんのナンバリングに合わせて行います。

そのためにまず、ナンバリングを登録しましょう。「概論」では、ネット上で公開されているナンバリングのページを紹介しました。ここでは、それに筆者のナンバリングを加えた5つのナンバリングを列挙してあります。この中から1つ選んで、「ナンバリング登録ページ」のリンク先で「保存」を押してください。リンク先はナンバリングが埋まった状態で表示されますが、「保存」ボタンを押すまでは保存されていませんので忘れないようにしてください。

ナンバリング登録はできましたね？ 次はキューブの面の色を登録してください。私はスクランブルする時と同じように白上・緑前の向きでキューブを持ってソルブしていますが、どのように持つかは自由です。開眼3x3x3でクロスを作る面を下にすると、色の関係性が頭に入っているのでナンバリングとの対応を覚えやすいかもしれませんね。

ナンバリングと面の色の設定ができたら、設定を反映させるためにこのページを再読み込みしてください。

例を通して分析を学ぶ - エッジ

ここからは、具体的なスクランブルを例にして、どのように分析を行うのかを説明します。

1回のループで終わる場合

まずは、次のスクランブルを回して、実際に崩してみてください。このキューブは、分析結果の文字列が2文字になる状態です。以下、スクランブルはさんが設定した、「U面: , F面: 」の向きで行い、これからの分析もそのまま同じ向きで行ってください。前節の繰り返しになりますが、もし基準面が違う場合は、キューブの面の色を登録してから、このページを再読み込みしてください。

(U' L' U M2 U' L U),
(U R U' M2 U R' U')

ここで、始める前に1つルールを説明しておきます。エッジのステッカーを表すためにDFのようなアルファベットで表現します。これは、「D面とF面の間のエッジパーツの、D面のステッカー」という意味です。同じパーツでも、DFとFDは違うステッカーを意味するので、注意してください。

このスクランブルをさんのナンバリングに従って分析すると、「」となります。(このような分析結果の文字列を、本記事では「分析文字列」と呼ぶことにします。)

分析は次のようなステップで行います。エッジの分析はDFのステッカーから始めます。これはこいういうものとして覚えてしまってください。なお、DFステッカーをバッファと呼ぶので、これも覚えておきましょう。

  1. バッファのDFステッカーを見る → (と)だ → これが本来あるべき位置はFR(「」)だ
  2. FRを見る → (と)だ → これが本来あるべき位置はFL(「」)だ
  3. FLを見る → (と)だ → これが本来あるべき位置はDF(バッファ)だ
  4. 開始点に戻ってきた。他のパーツは、全て正しく埋まっている → 分析完了だ
赤文字をつなぎ合わせたものが、分析文字列です。

1回のループで終わらない場合

今見たスクランブルは、分析のループが1周で終わりました。次は、ループが続く場合をやってみましょう。

(M2),
(U' L' U M2 U' L U),
(M2),
(U R U' M2 U R' U')

分析文字列は、途中でまだ分析していないパーツを自由に選べるため、4パターンあります。どれでも大丈夫ですが、ここでは最初の分析文字列を例に説明します。

  • 「 」
  • 「 」
  • 「 」
  • 「 」
同じく、赤文字をつなぎ合わせたものが、分析文字列です。

分析のステップは、次の通りです。

  1. DFを見る → (と)だ → これが本来あるべき位置はFR(「」)だ
  2. FRを見る → (と)だ → これが本来あるべき位置はDF(バッファ)だ
  3. 分析していないパーツは無いか？ あった！ UBのパーツとFLのパーツだ
  4. UBとFLの2つのパーツに属する4つのステッカーのうち、どれでもいいので1つ選ぼう
    • ※F面 > U面 > R面…など、ステッカーを選ぶ優先順位をおおまかに決めておくとよい
    例えば、FL(「」)を選んだ場合
  5. FLを見る → (と)だ → これが本来あるべき位置はUB(「」)だ
  6. UBを見る → (と)だ → これが本来あるべき位置はFL(「」)だ
  7. ループの開始点であるFLのパーツに属するステッカーに戻ってきた → 他に分析していないパーツは無い → 分析完了だ

バッファが埋まっている場合

スクランブル: D' F2 R2 U2 R2 D R2 U2 B2 U R B2 R' U2 B2 R' U2 R'
逆スクランブル: R U2 R B2 U2 R B2 R' U' B2 U2 R2 D' R2 U2 R2 F2 D

コーナーも変化していますが、ここでは無視してエッジの分析だけを考えます。このスクランブルは、今まで見てきたスクランブルとは違って、バッファが属するDFパーツに正しいパーツが収まってしまっています。このような場合は、「既に最初のループ(0手で)完成している」と見なし、分析していないパーツを探すステップから開始すれば大丈夫です。

分析文字列は4パターンあります。

  • 「 」
  • 「 」
  • 「 」
  • 「 」
  1. バッファのDFを見る → 既に埋まっている
  2. 分析していないエッジパーツは無いか？ あった！ FRのパーツとFLのパーツだ
  3. FRとFLの2つのパーツに属する4つのステッカーのうち、どれでもいいので1つ選ぼう
    4. 例えば、FR(「」)を選んだ場合
  4. FRを見る → (と)だ → これが本来あるべき位置はFL(「」)だ
  5. FLを見る → (と)だ → これが本来あるべき位置はFR(「」)だ
  6. ループの開始点であるFRのパーツに属するステッカーに戻ってきた → 他に分析していないエッジパーツは無い → 分析完了だ

なお、このようにパート内の分析文字列が奇数文字の時には、「パリティ処理」という手順が必要となります。パリティ処理については「実行編」の記事でご説明します。

EO

エッジパーツの位置が同じだが向きが違う場合をEOと呼びます(edges orientedの略)。EOについては、専用手順を使うとスムーズですが、ここでは覚える必要のある手順を少なくするために、2文字として捉える方法を紹介します。

スクランブル: R2 U2 R2 D R2 D' R2 U' F2 U' F' R U F2 U' F' R'
逆スクランブル: R F U F2 U' R' F U F2 U R2 D R2 D' R2 U2 R2

今までから見ると変に感じるかもしれませんが、同じ方法で分析することができます。

  • 「 」
  • 「 」
  1. DFを見る → 既に埋まっている
  2. 分析していないエッジパーツは無いか？ あった！ FRのパーツだ
  3. FRのパーツに属する2つのステッカーのうち、どれでもいいので1つ選ぼう
    4. 例えば、FR(「」)を選んだ場合
  4. FRを見る → (と)だ → これが本来あるべき位置はRF(「」)だ
  5. ループの開始点であるFRのパーツに属するステッカーに戻ってきた → 他に分析していないエッジパーツは無い → 分析完了だ

例を通して分析を学ぶ - コーナー

さて、次はコーナーです。 コーナーも、始める前に1つルールを用意しておきます。コーナーのステッカーを表すためにUBLのようなアルファベットで表現します。これは、「U面とB面とL面にまたがるコーナーパーツの、U面のステッカー」という意味です。最初の1文字で注目するステッカーを表し、2文字目と3文字目はアルファベット順に並べます。(2文字目と3文字目の順番は一般的には自由なのですが、本記事では順番を決めて曖昧性をなくしておきます。) コーナーでは、UBLステッカーがバッファ、つまり分析の出発点となります。

1ループで終わる場合

スクランブル:

R (U' R' U' R U R' F' R U R' U' R' F) R,
(U' R' U' R U R' F' R U R' U' R' F) R2

分析文字列は、「」となります。

  1. バッファのUBLステッカーを見る → (とと)だ → これが本来あるべき位置はRFU(「」)だ
  2. RFUを見る → (とと)だ → これが本来あるべき位置はRDF(「」)だ
  3. RDFを見る → (とと)だ → これが本来あるべき位置はUBL(バッファ)だ
  4. 開始点に戻ってきた。他のパーツは、全て正しく埋まっている → 分析完了だ

1ループで終わらない場合

スクランブル:

R (U' R' U' R U R' F' R U R' U' R' F) R,
R2 (U' R' U' R U R' F' R U R' U' R' F) ,
R (U' R' U' R U R' F' R U R' U' R' F) R,
(U' R' U' R U R' F' R U R' U' R' F) R2

  • 「 」
  • 「 」
  • 「 」
  • 「 」
  • 「 」
  • 「 」
  1. バッファのUBLステッカーを見る → (とと)だ → これが本来あるべき位置はRFU(「」)だ
  2. RFUを見る → (とと)だ → これが本来あるべき位置はUBL(バッファ)だ
  3. 分析していないパーツは無いか？ あった！ RDFのパーツとRBDのパーツだ
  4. RDFとRBDの2つのパーツに属する6つのステッカーのうち、どれでもいいので1つ選ぼう
    5. 例えば、RDF(「」)を選んだ場合
  5. RDFを見る → (とと)だ → これが本来あるべき位置はRBD(「」)だ
  6. RBDを見る → (とと)だ → これが本来あるべき位置はRDF(「」)だ
  7. ループの開始点であるRDFのパーツに属するステッカーに戻ってきた → 他に分析していないパーツは無い → 分析完了だ

バッファが埋まっている場合

スクランブル: D' F2 R2 U2 R2 D R2 U2 B2 U R B2 R' U2 B2 R' U2 R'
逆スクランブル: R U2 R B2 U2 R B2 R' U' B2 U2 R2 D' R2 U2 R2 F2 D

このスクランブルは、エッジのときと同じです。今回は、エッジは無視してコーナーだけを分析しましょう。しかし、よく見てみると、このコーナーの崩れ方は、1ループで終わらない場合のスクランブルでの、ステップ3からと同様です。現在のスクランブルと見比べながら、読み直してみてください。

CO

コーナーパーツの位置が同じだが向きが違う場合をCOと呼びます(corners orientedの略)。COについても、EOと同様に専用手順を使うとスムーズですが、ここでは2文字として捉える方法を紹介します。

スクランブル: (L' U2 L U L' U L), (R U2 R' U' R U' R')
逆スクランブル: (R U R' U R U2 R'), (L' U' L U' L' U2 L)
